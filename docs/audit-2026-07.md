# Holistic audit — July 2026

A twelve-dimension read of the codebase (architecture, durability, logic, UI/
a11y, programme content, performance, dependencies/config, storage schema,
efficiency/dead-code, security, PWA/offline, repo redundancy). Findings were
verified against the real code before landing here; the raw per-dimension
reports are archived in the session transcript.

**The headline is not sixty independent bugs — it is three systemic roots plus
two smaller ones.** Most P0/P1s hang off one of: (1) the sync payload is a
monolith, (2) auth was never actually built, (3) local-vs-UTC date math was
unsettled. The count below is deduped across dimensions (the finalise bug, for
instance, was independently found by durability AND architecture — it is one
line here).

## Deliberate decisions on the record

- **Reads/writes stay ungated (#20, #21).** No sensitive training data; the passkey lock's job is the destructive-op gate, not confidentiality. Revisit only if the data model gains something private.
- **Auth = reinforce, not rebuild.** No account system; passkeys remain an optional lock, now cryptographically real (#221).
- **Sync monolith (#2) is a design note, not a patch.** Third fix in the same territory → the next deliverable is a windowing/delta-sync design doc, per house rule.
- **Accessibility cluster (#44–#53) — deferred by decision (2026-07-15).** Forge is a deliberately visual-first app; strong-visual-impairment users are not an expected reliance case, so the interaction-layer a11y work (keyboard operability, focus traps, aria-live, focus ring, touch-target sizing, pinch-zoom) is parked, not planned. Checked against the one stated caveat — organic discovery — and confirmed SEO-neutral: discovery runs on the static library pages (semantic content + JSON-LD, audited clean), and none of the deferred items are crawlable content or ranking signals (Google doesn't rank on the Lighthouse a11y score; CWV/performance is clean). Revisit only if the audience assumption changes.
